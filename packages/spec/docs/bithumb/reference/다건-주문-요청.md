# 다건 주문 요청

여러 건의 주문을 한 번의 요청으로 생성합니다. 요청당 최대 20건까지 가능합니다.

> 예시코드는 JavaScript, Python, JAVA에 한해서만 제공합니다.

<Callout icon="📌" theme="warning">
  #### 입력 규칙

  * 지정가 주문: price와 volume 모두 필요
  * 시장가 매수: price(총 금액) 필요
  * 시장가 매도: volume 필요
  * `client_order_id` 는 회원 단위로 고유해야 합니다. 동일 요청(`batch_orders`) 내에서 중복된 ID가 포함된 경우, 최초 항목만 정상 접수되며 이후 중복 건은 실패 처리됩니다.
</Callout>

<Callout icon="📘" theme="info">
  #### 안내 사항

  * 서버가 요청을 정상적으로 수신 및 처리한 경우, 개별 주문 중 일부가 실패하더라도 상태 코드는 200 OK를 반환합니다.
  * 현재 API 호출 제한은 <span style={{color: "#dd1e2e"}}><strong>IP별 초당 20회</strong></span>로 설정되어 있습니다. 다만, 시스템 상황이나 트래픽에 따라 이 제한은 변경될 수 있습니다.
  * 기존에는 자전거래 가능성이 있을 경우 주문이 접수되지 않았으나, 개선 후에는 주문 접수 후 체결 단계에서 자전거래를 감지해 충돌하는 신규 주문 수량만 취소하도록 **개선될 예정**입니다. 자세한 내용은 [공지사항](https://apidocs.bithumb.com/changelog/%EC%97%85%EB%8D%B0%EC%9D%B4%ED%8A%B8-open-api-%EC%A3%BC%EB%AC%B8-%EA%B4%80%EB%A0%A8-%EC%9E%90%EC%A0%84%EA%B1%B0%EB%9E%98-%EB%B0%A9%EC%A7%80-%EC%8B%9C%EC%8A%A4%ED%85%9C-%EA%B3%A0%EB%8F%84%ED%99%94-%EC%95%88%EB%82%B4)을 참고하세요(업데이트 일시: 2026년 4월 2일 오후 12시 이후).

    응답 항목 "자전거래 방지 유형" `stp_type`이 신규 추가될 예정입니다.
</Callout>

***

#### 응답 항목 구분

`batch_orders_response` 배열 내 각 항목(Object)은 주문 처리 결과에 따라 성공 객체 또는 실패 객체 형태로 반환되며, 포함되는 필드가 다릅니다.

<Columns layout="auto">
  <Column>
    ✅ 성공 객체에 포함되는 필드

    * client\_order\_id(요청 시 전달한 경우에만)
    * order\_id
    * market
    * side
    * order\_type
    * created\_at
  </Column>

  <Column>
    ❌ 실패 객체에 포함되는 필드

    * client\_order\_id(요청 시 전달한 경우에만)
    * name
    * message
  </Column>
</Columns>

#### 성공/실패 구분 기준

* `order_id` 필드가 존재하면 **성공**
* `order_id` 필드가 없고 `name`, `message`가 존재하면 **실패**
