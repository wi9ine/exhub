# 주요 용어 안내

업비트 개발자 센터에서 사용되는 기본 용어들의 정의와 설명을 제공합니다.

# 거래소 서비스 관련 용어

### 페어(Trading Pair)

업비트에서 지원하는 거래 쌍. 오더북 운영의 단위입니다. 매수 또는 매도의 대상이 되는 자산(기준 자산, Base Asset)과 해당 자산을 매수하기 위해 지불하는 교환 자산(호가 자산, Quote Asset)의 쌍으로 이루어집니다. 코드는 {Quote Asset}-{Base Asset}의 형식으로 표기합니다.

(예시) KRW-BTC(원화로 비트코인을 매수, 비트코인을 매도하여 원화로 교환)

(예시) BTC-ETH(비트코인으로 이더리움을 매수, 이더리움을 매도하여 비트코인으로 교환)

<br />

### 마켓(Market)

동일한 호가 자산(Quote Asset)을 사용하는 페어들의 집합. 업비트는 원화(KRW) 마켓, BTC 마켓, USDT 마켓을 운영하고 있습니다. 원화를 Quote Asset으로 하는 모든 거래 쌍들을 원화 마켓으로 지칭합니다.

<br />

### 통화(Currency)

단일 자산의 통화를 지칭하기 위해 사용되는 용어입니다. 원화(KRW)와 같은 법정 화폐와 비트코인, 이더리움과 같은 디지털 자산을 모두 지칭할 수 있습니다. '통화 코드(Currency Code)'는 `KRW`, `BTC`, `ETH` 와 같은 코드 문자열 값을 지칭하기 위해 사용합니다.

<br />

# 시세 관련 용어

### 캔들(OHLCV)

페어에 특정 기간 동안 발생한 거래들에 대해 시가(Opening Price), 고가(Highest Price), 저가(Lowest Price), 종가(Closing Price), 거래량(Volume)을 산출한 통계 데이터입니다. 주로 차트 시각화를 위한 데이터로 사용되며, 집계 구간의 크기와 단위에 따라 초 캔들/분 캔들/일 캔들/주 캔들/월 캔들/연 캔들 등으로 구분합니다. 국문 표기 시 '봉'으로 지칭하기도 하며 초봉/분봉/일봉/주봉/월봉/연봉과 같은 용어와 혼용합니다. 분 캔들의 경우 세부 단위에 따라 3분봉, 10분봉과 같이 부르기도 합니다.

<br />

# 주문 관련 용어

### 주문(Order)과 체결(Trade)

**주문**은 사용자가 디지털 자산을 매수하거나 매도하기 위해 거래소에 제출하는 요청으로, 페어, 가격, 수량, 주문 유형(지정가, 시장가 등), 거래 옵션 등을 지정할 수 있습니다.

**체결**은 하나의 주문이 다른 사용자의 반대 주문과 매칭되어 전체 또는 일부 거래가 실행된 것을 의미합니다. 매수 주문과 매도 주문이 가격과 수량 조건에 맞춰 서로 일치하면 체결이 발생하며, 이때 실제 자산의 이동이 발생합니다.

주문은 반대 주문 현황에 따라 여러 번 나누어 체결될 수 있으며, 이에 따라 특정 시점에서는 일부 수량만 체결된 **부분 체결** 상태로 존재할 수 있습니다. 부분 체결 상태인 경우 체결되지 않은 수량을 **주문 잔량**으로 지칭합니다.

<br />

### 지정가 주문, 시장가 주문, 최유리지정가 주문

주문 생성 시 선택할 수 있는 주문 유형입니다. 각 주문 유형에 대한 간단한 설명은 아래와 같으며, 더 자세한 내용은 [주문 생성](https://docs.upbit.com/kr/reference/new-order) API 문서를 참조하시기 바랍니다.

* **지정가 주문**: 단가와 수량을 모두 지정하는 주문입니다.
* **시장가 주문**: 현재 반대 호가 중 가장 유리한 가격 조건으로 지정한 총액만큼 매수, 또는 지정한 수량을 매도하는 주문입니다.
* **최유리 지정가 주문**: 현재 최고 매수 호가 또는 최저 매도 호가와 동일한 단가로, 지정한 총액 또는 수량만큼 매수/매도하는 주문입니다.

<br />

### 스프레드(Spread)

최우선 매수호가(가장 높은 매수 가격)와 최우선 매도호가(가장 낮은 매도 가격) 간의 가격 차이를 의미합니다.

<br />

### 메이커(Maker)와 테이커(Taker)

거래소 오더북에 새로운 호가를 등록하는 주문을 메이커 주문, 오더북에 존재하는 반대 호가 주문에 즉시 체결하는 주문을 테이커 주문이라고 합니다. 쉬운 예시로, 메이커는 지정가 주문 등을 통해 현재가보다 낮거나 높은 가격에 매수/매도하는 오더북 주문을 생성(Make)하는 경우에 해당하며, 테이커는 시장가 주문 등을 통해 오더북에 존재하는 상대 호가 주문을 가져가는(Take)하는 경우에 해당합니다.

<br />

### Post Only

주문 생성 시 해당 주문이 오직 메이커 주문으로만 생성되는 것을 보장하기 위한 주문 옵션입니다. 테이커 주문으로서 즉시 체결될 가능성이 있는 경우 해당 주문의 생성을 자동으로 취소합니다.

<br />

### 자전 거래 체결 방지(SMP)

동일한 사용자로부터 생성된 매수 주문과 매도 주문이 상호 체결되는 것을 방지하기 위한 옵션입니다. API 지원 사항 등 자세한 내용은 [자전거래 체결 방지(Self-Match Prevention, SMP)](https://docs.upbit.com/kr/docs/smp) 페이지를 참고해주시기 바랍니다.
